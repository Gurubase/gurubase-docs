## Installation Options

You can use Gurubase in two ways:

1. **[Gurubase.io](https://gurubase.io)** - Start using immediately without any setup
2. **Self-hosted** - Deploy on your own infrastructure

## Self-hosted Installation

### Prerequisites

Before installation, ensure you have:
- [Docker](https://docs.docker.com/get-docker/) `27.3.x` or later.
- [Docker Compose](https://docs.docker.com/compose/install/) (`docker compose` or `docker-compose`) `2.30.x` or later.
- OpenAI API key (for answer generation and embeddings). Get it from [here](https://platform.openai.com/api-keys).

### Quick Install

Run this command to install Gurubase:

```bash
curl -fsSL https://raw.githubusercontent.com/Gurubase/gurubase/refs/heads/master/gurubase.sh -o gurubase.sh
bash gurubase.sh
```

The installer will:
1. Create a `.gurubase` directory in your home folder
2. Download and start all necessary services
3. Open the web interface at `http://localhost:8029`

### System Requirements

- **Supported OS**: Linux and macOS (Windows via WSL2)
- **CPU**: 4 cores minimum
- **RAM**: 8GB minimum
- **Storage**: 10GB SSD
- **Ports**: 8028 and 8029 must be available

<Note>
  Only Linux and macOS are supported. For Windows users, please use WSL2.
</Note>

## Architecture Overview

Gurubase follows a microservices architecture designed for scalability and performance. Here's a quick overview of the key components:

<AccordionGroup>
  <Accordion icon="layer-group" title="Core Components">
    - **Frontend**: Next.js application for the user interface
    - **Backend**: Django REST framework for API and business logic
    - **Vector Store**: Milvus for similarity search and embeddings
    - **Message Queue**: RabbitMQ for task distribution
    - **Cache**: Redis for rate limiting and session management
    - **Database**: PostgreSQL for application data
  </Accordion>
  
  <Accordion icon="gears" title="Key Features">
    - **RAG System**: Retrieval Augmented Generation for accurate answers
    - **Multi-source Processing**: Support for websites, PDFs, YouTube videos and GitHub repositories
    - **Embedding Options**: OpenAI or custom embedding API
    - **Trust Scoring**: LLM-based context relevance evaluation
    - **Binge Sessions**: Visualize your learning path while talking with a Guru. You can navigate through it and create a personalized path
  </Accordion>
</AccordionGroup>

<Note>
  For detailed architecture documentation, including system diagrams, data flows, and configuration options, see our [Architecture Guide](https://github.com/Gurubase/gurubase/blob/master/src/gurubase-backend/ARCHITECTURE.md) on GitHub.
</Note>
